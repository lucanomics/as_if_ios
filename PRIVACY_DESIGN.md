# PRIVACY_DESIGN — 개인정보 보호 설계

DeskShield는 "기록을 많이 남기는 도구"가 아니라 "개인정보를 남기지 않으면서 안내 품질을 관리하는
도구"입니다. 설계 전반에서 **비식별**과 **로컬 저장**을 최우선 원칙으로 삼습니다.

## 1. 개인정보를 저장하지 않도록 설계된 이유

창구 업무 특성상 민원인의 이름·등록번호·여권번호·연락처 등이 오갈 수 있습니다. 이런 정보가 업무기록에
남으면 그 자체가 개인정보 파일이 되어, 유출·오남용·과잉수집 위험이 생깁니다. DeskShield는 "무엇을
안내했는가"라는 **업무 패턴**만 비식별로 남기고, "누구였는가"는 남기지 않습니다.

## 2. Vercel 배포 시에도 데이터는 브라우저 로컬에만 저장됩니다

- 앱은 정적 프론트엔드로만 배포됩니다. 서버리스 함수, API Route, Edge Function, 클라우드 DB가 없습니다.
- 모든 업무 데이터는 브라우저 `localStorage`(추후 IndexedDB)에 저장됩니다.
- 어떤 사용자 입력도 네트워크로 전송되지 않습니다. 소스에는 `fetch`/`XHR`/`sendBeacon`/외부 API 호출이
  전혀 없으며, 런타임 네트워크 검사에서도 외부 요청 0건을 확인했습니다.
- 외부 폰트/아이콘/이미지 CDN, 원격 분석, 텔레메트리, 외부 에러 리포팅을 사용하지 않습니다.

## 3. 저장 금지 정보 목록

이름 · 외국인등록번호 · 여권번호 · 주민등록번호 · 생년월일 · 전화번호 · 이메일 · 주소 · 회사명 원문 ·
학교명 원문 · 학원명 원문 · 예약번호 전체 · 접수번호 · **실제 번호표 순번** · 서류 사진 · 전산 화면 캡처 ·
담당 직원 실명 비난 기록 · 민원인을 특정할 수 있는 상세 사건 서술.

**의심 패턴 자동 감지**(`src/lib/privacyGuard.ts`): 이메일, 전화번호, 7자리 이상 연속 숫자,
외국인등록번호/주민번호 유사 패턴, 여권번호 유사 패턴(영문+숫자), 주소 표현, 번호표 순번 표현
(예: `A-103`, `B27`, `8번 42번`), 기관명 원문 가능성("주식회사", "학원", "학교", "대표", "원장" 등).
감지 시 저장 전 경고하고 기본적으로 저장을 차단합니다.

## 4. 국적 필드 사용 시 주의사항

- 국적은 **민원인 식별 목적이 아니라 업무 패턴 분석용 비식별 분류값**입니다.
- 기본값은 "미기재"이며, 드롭다운/검색 선택 중심으로 입력합니다. 내부 저장값은 ISO 3166-1 alpha-2
  코드입니다.
- 국적 + 체류자격 + 날짜 + 특이 사건 + 기관명 조합은 개인 식별 위험을 만들 수 있습니다. 소수 국적이나
  특이 사건이 있는 경우 메모에 구체 정보를 적지 마세요(앱 내 경고 표시).

## 5. 번호표 유형 필드 사용 시 주의사항

- 번호표는 **유형만** 비식별로 기록합니다(미부여/비예약/예약 확인/사증/8번/7번/국적/일반 안내/담당자
  인계 등).
- **실제 번호표 순번·호출번호·접수번호는 저장하지 않습니다.** 순번처럼 보이는 입력은 개인정보 의심
  패턴으로 감지됩니다.

## 6. 비식별 기록 원칙

- 자유 입력보다 칩 버튼·프리셋·자동 템플릿을 우선합니다.
- 메모 없이도 의미 있는 로그가 완성되도록 설계했습니다(메모 입력란은 기본 접힘 + 경고 문구).
- 완벽한 기록보다 누락 방지를 우선하며, 미완성 저장을 정상 플로우로 인정합니다.
- 위험 민원은 자동 판정하지 않고 "담당자 확인 필요"로 유도합니다.

## 7. 로컬 저장 원칙

- 저장소는 `src/lib/storage.ts`의 추상화 계층을 통해 접근합니다. 기본은 **IndexedDB**(`src/lib/idb.ts`)이며,
  사용 불가 시 localStorage로 폴백합니다. 기존 localStorage 데이터는 IndexedDB로 **1회 자동 이관**되고, 이관 후
  localStorage의 업무 데이터 사본은 제거되어 데이터가 두 곳에 남지 않습니다.
- 수정 이력(`AuditHistory`)은 **변경된 필드명만** 기록하고 값(before/after)은 저장하지 않아, 개인정보가
  이력에 남지 않도록 했습니다.
- 보존 기간(기본 90일)을 두고, 자동 삭제는 사용자의 명시적 확인 후에만 실행합니다.
- **로컬 앱 잠금(PIN)**: PIN 평문을 저장하지 않고 Web Crypto PBKDF2(150k 반복, per-user salt)로 파생값만
  저장합니다. 잠긴 동안에는 불투명 화면으로 덮어 업무 데이터가 노출되지 않습니다.
- **PWA 서비스 워커**는 same-origin 정적 자산만 프리캐시하며, 업무 데이터(IndexedDB/localStorage)는 캐시하지
  않고 외부(cross-origin) 요청은 가로채지 않습니다. 런타임 외부 네트워크 호출은 0건입니다.

## 8. 내보내기 파일 취급 주의사항

- 내보내기 전 전체 로그를 다시 스캔하며, 의심 패턴이 있으면 강한 경고 후에만 진행할 수 있습니다.
- 내보낸 파일은 이 기기에만 생성되며 어디로도 전송되지 않습니다.
- 외부 클라우드·메신저·공용 PC에 저장하지 마세요.

## 9. 공식 행정처리 시스템이 아님

DeskShield는 개인용 비식별 안내 품질관리 도구입니다. 공식 행정처리 기록이 아니며, 법률 자문·허가 판정·
민원인 추적 도구가 아닙니다. 앱 하단과 설정 화면, 첫 실행 화면에 이 고정 문구를 항상 표시합니다.
