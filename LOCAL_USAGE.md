# LOCAL_USAGE — 로컬 사용 안내

## 1. 로컬에서 실행하는 방법

```bash
npm install
npm run dev      # http://localhost:5173
```

빌드 결과를 확인하려면:

```bash
npm run build && npm run preview   # http://localhost:4173
```

첫 실행 시 배포/개인정보 경고 온보딩이 표시됩니다. "이해했습니다. 시작하기"를 눌러 시작합니다.

## 2. 업무 중 빠른 기록 방법 (10초 목표)

1. 우측 상단 **`+ 빠른 기록`** 클릭(또는 `⌘/Ctrl + N`).
2. 필요하면 상단 **프리셋**을 눌러 대부분의 값을 한 번에 채웁니다.
3. 칩 버튼으로 필수 5개를 선택합니다: ① 체류자격 ② 국적 ③ 민원유형 ④ 안내범위 ⑤ 번호표 유형.
4. **저장 (요약 확인)** → 한 줄 요약을 확인하고 **확정 저장**.
   - 같은 유형이 이어지면 **저장 + 같은 유형 계속**(`⌘/Ctrl + Shift + Enter`).
   - 나중에 정리할 건은 **저장 + 나중에 보완**(Review Queue로 이동).
5. 저장 직후 토스트에서 **수정 / 복제 / 같은 유형 계속 / 닫기** 로 바로 조치할 수 있습니다.

메모는 선택입니다. 필요할 때만 "메모 펼치기"로 열고, 개인정보를 적지 않습니다.

## 3. 오늘 근무 세션 사용 방법

1. **근무 세션** 탭 → "오늘 근무 세션"에서 오늘 자주 쓰는 기본값(체류자격/국적/민원유형/번호표/안내범위/
   안전문구)을 설정하고 **오늘 기본값 저장**.
2. 이후 빠른 기록을 열면 이 기본값이 자동으로 채워집니다.
3. 기본값은 앱을 재시작해도 **당일에는 유지**되고, 다음 날에는 자동 초기화됩니다.

## 4. 퇴근 전 3분 리뷰 방법

**근무 세션** 탭의 "퇴근 전 3분 리뷰" 카드에서 오늘의 지표를 한눈에 봅니다:
총 로그 · 미완성 · 위험 키워드 · 위험 조합 · 안전문구 미사용 · 개인정보 의심 · 담당자 인계 미기재 ·
비예약 번호표. 수치가 있는 항목은 **Review Queue** 또는 **개인정보 청소 모드**에서 바로 보완합니다.

## 5. 개인정보 청소 모드 사용 방법

1. **개인정보 청소** 탭으로 이동.
2. 상단 필터(전체/숫자 과다/이메일/전화번호/여권번호/기관명/주소/번호표 순번)로 의심 로그를 좁힙니다.
3. 각 로그에서 **메모 비우기 / 비식별 표현으로 수정 / 문제 없음으로 표시 / 로그 삭제**.
4. 내보내기 전, 그리고 퇴근 전 리뷰 시 이 모드를 한 번씩 확인하세요.

## 6. 백업 방법

1. **내보내기** 탭으로 이동(내보내기 전 자동 재스캔).
2. 의심 패턴이 없으면 **JSON 내보내기** 또는 **CSV 내보내기**.
3. 복원은 같은 화면의 **JSON 가져오기**(기존 로그와 병합, id 중복 제외).

> 백업 파일은 이 기기에만 생성됩니다. 외부 클라우드·메신저·공용 PC에 올리지 마세요.

## 7. 실수로 개인정보를 입력했을 때 삭제하는 방법

**개인정보 청소** 탭 하단 **긴급 삭제** 영역:

- 특정 로그의 **메모만 비우기** 또는 **로그 삭제**
- **오늘 작성한 로그 전체 삭제**
- **모든 로컬 데이터 초기화**

모든 삭제는 확인 모달을 거치며, 삭제 후에는 되돌릴 수 없습니다.
